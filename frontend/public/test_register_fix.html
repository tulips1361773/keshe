🔧 注册功能修复测试

问题描述

用户在注册时遇到 "该字段是必填项" 错误，原因是前端发送的确认密码字段名为 confirmPassword，而后端期望的字段名为 password_confirm。

修复方案

在前端注册数据中添加 password_confirm 字段，将 confirmPassword 的值映射过去。

测试用例 1: 正常注册数据
{ "username": "testuser123", "real_name": "测试用户", "phone": "13800138000", "email": "test@example.com", "password": "Abc12345!", "password_confirm": "Abc12345!", "user_type": "student" }
测试用例 2: 包含扩展特殊字符的密码
{ "username": "testuser456", "real_name": "测试用户2", "phone": "13800138001", "email": "test2@example.com", "password": "Abc12345_", "password_confirm": "Abc12345_", "user_type": "student" }
测试用例 3: 密码不匹配
{ "username": "testuser789", "real_name": "测试用户3", "phone": "13800138002", "password": "Abc12345!", "password_confirm": "Abc12345@", "user_type": "student" }
